# PHASE 4 — Looker Studio 대시보드 설계

> 본문에서 사용하는 약어·게임 용어 정의는 [README §4 — 본문에 자주 등장하는 용어](../README.md#본문에-자주-등장하는-용어)를 참고.

## 0. 목적

분석 결과를 5분 안에 시청자가 따라올 수 있는 형태로 전달하는 것이 목적이다.
**3쪽 구성** — 한 페이지 한 메시지, 페이지 상단에 결론 한 줄.

## 1. 데이터 소스 (outputs/, Looker Studio 연결용 깨끗한 5종)

| 파일 | 단위 | 주요 컬럼 |
|---|---|---|
| `champion_meta.csv` | patch × champion × role | pick_count · pick_rate_pct · win_rate_pct · wilson_low_pct · **flag** · **riot_15_3_action** |
| `diversity.csv` | patch × role | unique_champs · hhi · effective_champs · gini |
| `patch_drift.csv` | champion (한 챔프 한 행) | n_15_1 · n_15_3 · pickrate_diff_pp · pr_ci_low/high_pp · winrate_diff_pp · wr_ci_low/high_pp · **riot_15_3_action** · enough_sample |
| `player_segments.csv` | 게임 단위 (55,840행) | patch · cluster · persona · gpm · dpm_champ · vision · kp · (참고) kda · deaths |
| `segment_summary.csv` | cluster (4행) | persona · n_games · share_pct · win_rate_pct · 평균 + centroid z |

> 모든 챔프 표는 `champion` (한글) 컬럼을 차원으로, `champion_en` (영문)을 join key.
> `riot_15_3_action` 컬럼은 라이엇 V15.3 패치 노트의 실제 조정 (NERF / BUFF / ADJUSTMENT / 빈 칸).

## 2. 페이지 1 — "메타가 어떻게 움직였는가" + ⭐ 패치노트 검증

### 페이지 한 줄 (상단 배너)
> **"15.3 메타는 챔프 풀이 넓어졌으나(unique champs ↑) 신규 픽은 대부분 비주류 → 다양성 지표는 long tail 쪽으로 이동. 분석이 잡은 NERF·BUFF 신호는 라이엇 실제 패치 조정 7건과 일치, 무엇보다 15.1 시점에 발견한 사장된 강캐(스웨인)를 라이엇이 15.3에 실제 버프했다."**

### P1 의도된 시각적 메시지

분석가가 잡은 NERF 후보 옆에 라이엇 ✓ 매칭이 한 줄에서 보이면, 담당자는 별도 문서 안 봐도 *"분석 신호가 실전에서 작동했다"* 를 표 하나로 이해.

## 3. 페이지 2 — "교란 변수를 어떻게 통제했는가" ⭐⭐ 포폴 심장

### 페이지 한 줄
> **"티어 통제 없이 본 픽률 변화는 패치 효과를 최대 2.7배 부풀린다. 이즈리얼은 naïve 분석에서 떠오른 챔프 5위지만, 같은 티어 코호트(PLAT+EM+DIA)로 좁히면 30위로 떨어진다 — 픽률 상승의 대부분은 패치 효과가 아니라 표본의 티어 분포 차이였다."**

### P2 시연 흐름 (5분 자기소개)

1. **거대 KPI 박스**의 "5위 ↔ 30위 / 2.7배 과대평가"를 1초에 잡기.
2. *"왜 이런 일이 벌어지나?"* → 좌하 티어 분포로 **"15.1과 15.3은 같은 패치 비교가 아니라 다른 티어 코호트 비교에 가까웠다"** 설명.
3. *"그래서 어떻게 통제했나?"* → 우하 ranking 표.

## 6. Looker Studio 연결 순서

1. `outputs/` 의 5종 CSV 를 Google Drive 폴더에 업로드.
2. Looker Studio → 새 보고서 → 데이터 소스 5개 추가 (Google Sheets 또는 CSV upload).
3. 페이지 3장 작성:
   - **P1**: `champion_meta.csv`, `diversity.csv`, `patch_drift.csv`(상단 5건 필터)
   - **P2**: `patch_drift.csv` + 별도 cohort_vs_naive 표 (또는 `13_cohort_vs_naive_top.csv` 그대로 활용)
   - **P3**: `segment_summary.csv`, `player_segments.csv`, PNG 임베드
4. 보고서 수준 필터: `patch` (15.1 / 15.3 / 전체), `role` (5 + 전체).
5. P1·P2·P3 각 페이지 상단에 위 결론 한 줄을 텍스트 위젯으로 박기.
