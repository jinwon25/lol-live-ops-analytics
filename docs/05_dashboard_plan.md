## 2. 페이지 1 — "메타가 어떻게 움직였는가" (메타 헬스 + 패치 드리프트)

**한 줄 메시지**: *"15.1 → 15.3 패치 전환으로 챔프 풀은 넓어졌지만(uniq_champs ↑) 일부 라인은 쏠림이 강해졌다. 비에고·오공·미스 포츈은 라이엇 실제 너프와 픽률 하락이 정확히 매칭됐다."*

```
┌──────────────────────────────────────────────────────────────────┐
│ [상단 KPI 카드 4개]                                              │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐                 │
│  │ 15.1    │ │ 15.3    │ │ 코호트   │ │ NERF/BUFF│                 │
│  │ 2,073   │ │ 6,285   │ │ N=55,840│ │ 85 / 10  │                 │
│  │ games   │ │ games   │ │         │ │ 후보     │                 │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘                 │
├──────────────────────────────────────────────────────────────────┤
│ [좌] 떠오른 챔프 TOP 10 (코호트)        [우] 저문 챔프 TOP 10    │
│  가로 막대, 픽률 차(pp), 한글명          가로 막대                │
│  CI 에러바 표시 — 면접에서 "왜 진짜?"    오공/비에고/미스 포츈    │
│  바로 답 가능                                                     │
├──────────────────────────────────────────────────────────────────┤
│ [좌] role별 다양성 (HHI/Gini Δ)      [우] NERF 후보 (강력+인기)  │
│  5개 role × 2 metric 그룹 막대        Table — patch/role/챔프    │
│  BOT 단독 HHI ↑ 강조                  필터: patch · role           │
└──────────────────────────────────────────────────────────────────┘
```

## 3. 페이지 2 — "교란 변수를 어떻게 통제했는가" ⭐ 포폴 심장

**한 줄 메시지**: *"통제 없는 단순 픽률 변화는 패치 효과와 티어 분포 효과가 섞여 있다. 이즈리얼은 naïve 분석에서 떠오른 챔프 5위로 잡히지만, 같은 티어 구간(PLAT+EM+DIA)으로 통제하면 30위로 떨어진다 — 픽률 상승의 대부분이 패치 효과가 아니라 표본의 티어 분포 차이였다."*

```
┌──────────────────────────────────────────────────────────────────┐
│ [상단 거대 KPI]                                                  │
│   이즈리얼 — naïve rank 5위 ↔ 코호트 rank 30위                   │
│   naïve가 패치 효과를 2.7배 과대평가                              │
├──────────────────────────────────────────────────────────────────┤
│ [중앙 대비 차트] 가로 막대 한 쌍                                 │
│   ●━━━ 이즈리얼 — naïve +0.353 pp ━━━━━━━━━━━━━━━━●              │
│   ●━━━ 이즈리얼 — 코호트 +0.132 pp ━━━━━━●                       │
│   ●━━━ 야스오 — naïve −0.073 pp ━●                              │
│   ●━━━ 야스오 — 코호트 +0.261 pp ━━━━●  (반대 방향)             │
│   ●━━━ 퀸 — naïve +0.306 pp ━━━━━━━━●                            │
│   ●━━━ 퀸 — 코호트 (15위 밖, 표시 안 됨)                         │
├──────────────────────────────────────────────────────────────────┤
│ [좌] 솔로 티어 분포 (15.1 vs 15.3)    [우] 교란 효과 표           │
│   누적 막대 100%                      ranking 표 (cohort vs naïve)│
│   15.1 MASTER+ 22.9%, 15.3 7.0%       Δ rank, Δ pp                 │
│   왜 통제가 필요했는지 시각화         hover: 한글명 + 영문         │
└──────────────────────────────────────────────────────────────────┘
```

면접 시연 흐름:
1. KPI 박스의 "5위 ↔ 30위 / 2.7배 과대평가" 한 눈 강조
2. "왜 이런 일이?" → 좌하단 티어 분포 차트로 *"15.1과 15.3은 티어 코호트 자체가 다르다"* 설명
3. "그래서 어떻게 통제했나?" → 우하단 ranking 표 — 이즈리얼·야스오·퀸 세 케이스

## 4. 페이지 3 — "유저는 어떻게 나뉘는가" (세그멘테이션)

**한 줄 메시지**: *"같은 티어 안에서도 4개의 행동 프로필이 나타난다. 슈퍼 캐리형(19%, 승률 76%) vs 스노우볼 패배 프로필(27%, 승률 33%)이 양극, 시야 컨트롤형(20%) vs 전투 추격형(34%)이 중간 — 같은 승률대에서 스타일이 갈라진다."*

```
┌──────────────────────────────────────────────────────────────────┐
│ [페르소나 4개 카드]                                              │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ │
│  │ 시야         │ │ 전투        │ │ 스노우볼    │ │ 슈퍼        │ │
│  │ 컨트롤형     │ │ 추격형      │ │ 패배 프로필 │ │ 캐리형      │ │
│  │ 20.0%        │ │ 34.0%       │ │ 27.0%       │ │ 19.0%       │ │
│  │ win 52%      │ │ win 49%     │ │ win 33%     │ │ win 76%     │ │
│  │ 시야↑        │ │ KP↑ vis↓    │ │ 전반 ↓      │ │ GPM·DPM↑↑   │ │
│  └─────────────┘ └─────────────┘ └─────────────┘ └─────────────┘ │
├──────────────────────────────────────────────────────────────────┤
│ [좌] PCA 2D 산점도 이미지            [우] 페르소나별 핵심 지표   │
│  (`_clusters_pca_2d.png` 임베드)      방사형 차트 4 × 4 지표 (z) │
│                                                                  │
├──────────────────────────────────────────────────────────────────┤
│ [좌] 클러스터 × 역할 (heatmap)       [우] 15.1 → 15.3 비율 변화  │
│  20% 근처 균등 분포 — role 통제 검증   100% 누적 막대 2개         │
│  (`_cluster_role_heatmap.png`)        시야형 +3.1pp 변화 강조     │
```

## 6. Looker Studio 데이터 연결 순서

1. `outputs/` 폴더를 Google Drive 폴더에 업로드 (또는 GitHub raw URL → Sheets import → Looker Studio 데이터 소스).
2. 데이터 소스 11개 등록:
   - `10_champ_stats_cohort.csv` — Page 1 보조
   - `12_flagging_cohort.csv` + `12_flagging_summary.csv` — Page 1 우하단
   - `13_rising_champs_cohort.csv` + `13_falling_champs_cohort.csv` — Page 1 중앙
   - `13_diversity_drift_by_role.csv` — Page 1 좌하단
   - `13_cohort_vs_naive_top.csv` — Page 2 메인
   - `13_rising_champs_naive.csv` — Page 2 보조
   - `segment_centroids.csv` + `segment_role_distribution.csv` + `segment_patch_shift.csv` — Page 3
3. 한국어 챔프명은 `champion_name_kr` 컬럼을 표시 차원으로 사용 (영문 `champion_name`은 join key).
4. 페이지 간 필터: patch (15.1 / 15.3 / 전체), role (5 + 전체).
