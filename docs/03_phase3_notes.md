# PHASE 3 인사이트 메모 — 플레이 스타일 세그멘테이션 (축2)

## 0. 분석 설계 — 세 가지 핵심 선택과 그 이유

### 0.1 표본은 v_cohort(PLAT+EM+DIA)
전체 풀에서 군집화하면 마스터/그랜드마스터의 KDA·GPM이 골드보다 구조적으로 높으니 클러스터가 그냥 "티어 사다리"를 재현. 코호트로 좁히면 **같은 실력대 안에서 스타일이 어떻게 갈리나**가 드러남.

### 0.2 식별자 한계 → "플레이어"가 아니라 "플레이 스타일(게임 단위 행동 프로필)" 군집화
puuid는 P151에는 있지만 **P153에는 없음**. 같은 플레이어를 두 패치에 걸쳐 매칭할 수 없으므로 군집의 단위는 *플레이어*가 아니라 *한 게임에서 한 플레이어가 보인 행동 프로필*. 메모와 라벨에 이 한계를 정직하게 명시.

### 0.3 role 처리 — 역할 내 z-score 채택
세 옵션을 두고 검토:
| 옵션 | 결과 |
|---|---|
| (a) role 원핫 + 전체 표준화 | role 더미가 큰 분산 → 클러스터가 결국 포지션으로 갈림 |
| (b) role 제거 | SUPPORT의 낮은 GPM 같은 구조적 차이를 무시 → 정보 손실 |
| **(c) 역할 내 z-score** | 각 (role) 그룹의 mean·std로 표준화. "역할 안에서의 상대적 성향"만 군집화 |

**(c) 채택**. 결과 검증: 4개 클러스터 모두 5개 role을 거의 균등(17~22%)하게 포함 → role이 클러스터를 지배 안 함. 설계 의도대로 작동.

## 1. 핵심 분석가 판단 — 피처를 재선택한 과정

**1차 시도 (6피처)**: kda, gpm, dpm_champ, vision_score, kill_participation, deaths
- 결과: silhouette 최댓값 K=2, 두 클러스터 승률 **75% vs 33%**
- **즉 군집화가 "플레이 스타일"이 아니라 "이긴 게임 vs 진 게임"으로 수렴**
- 원인: KDA = (k+a)/d, deaths = 죽음 횟수 — 둘 다 **결과(win)의 강한 proxy**. 결과 신호를 입력으로 넣으면 K-means가 자연히 win 축으로 갈림

**2차 시도 (4피처, 프로세스 지표만)**: gpm, dpm_champ, vision_score, kill_participation
- 결과: 의미 있는 4개 페르소나
- 결과 신호(kda, deaths)는 사후 클러스터 해석용 reference로만 사용

> **이 두 단계 흐름이 PHASE 3의 핵심 어필 포인트.** silhouette만 보고 K=2를 받아들이지 않고, "왜 K=2인가?"를 진단해 피처 자체를 재설계한 것 = 분석가 판단의 증명.

## 2. K 선택 — 엘보우 + 실루엣

| K | inertia | silhouette |
|---|---|---|
| 2 | 152,976 | **0.271** ← max |
| 3 | 128,201 | 0.232 |
| **4** | **108,693** | **0.212** ← elbow 꺾임 |
| 5 | 97,784 | 0.206 |
| 6 | 89,927 | 0.193 |

silhouette 최댓값은 K=2지만 **(1차 시도와 같은 자명 분리 회피)** + **inertia 감소율이 K=4부터 둔화** → **K=4 채택**.

### 정직한 해석 — 군집-승률 상관에 대해

- KDA·deaths를 빼도 GPM·DPM·KP는 win과 어느 정도 상관(자원·딜·참여를 잘하는 사람이 결국 잘함). 그래서 **C2 위축·부진형(승률 33%)과 C3 슈퍼 캐리형(승률 76%)은 여전히 "결과의 자연스러운 반영"**.
- 그러나 **C0 시야 컨트롤형(52%)과 C1 팀합세형(49%)은 승률이 거의 같은 중간대** — 같은 성과인데 스타일이 다른 진정한 "스타일 분리". 이 두 군집의 존재 자체가 군집화의 가치.

## 6. 콘텐츠 / 리텐션 가설 — 라이브 운영 관점

> **전제**: 이 페르소나는 *플레이어 단위가 아니라 게임 단위 행동 프로필*. 즉 "이 사람이 위축형"이 아니라 "이 게임에서 위축된 행동을 보였다". 가설은 그 한계 위에서 작성.

### 가설 A — 위축·부진형(27%)에 대한 캐치업 메커니즘
- **관찰**: 모든 행동 지표가 ↓, 승률 33%. 게임당 1/4 정도가 "지고 있을 때 행동이 위축되는 패턴". 큰 풀.
- **가정**: 스노우볼 후 어떻게 해도 회복이 어렵다는 인식이 행동 위축 → 다음 게임 매칭 회피·이탈 위험.
- **콘텐츠 액션**: 후반 캐치업 골드/EXP 메커니즘 강화(이미 있는 처형 보상의 확장 등), 또는 **"패배해도 시도 보상"** — KP / 와드 임계 기반 일일미션. 결과 의존이 아닌 행동 의존 보상.
- **리텐션 액션**: 연속 부진 게임 후 휴식 권고 / 챔프 시도 모드(친선전) 추천 알림.

### 가설 B — 슈퍼 캐리형(19%)을 hardcore retention의 코어로
- **관찰**: 승률 76%, GPM·DPM·KP 모두 폭주. 가장 강한 코어 유저 후보군.
- **콘텐츠 액션**: 챌린지 모드(아레나, 1v1, 한정 챔프 챌린지), 챔프 마스터리 7급 이상의 상위 시스템, 시즌 한정 캐리 챔프 리워드.
- **리텐션 액션**: 멘토링 시스템 — 슈퍼 캐리형 유저를 위축·부진형 유저에 매칭(코치 보상). 게임 내 멘토 → 신규/회복 유저 리텐션도 같이 잡힘.

### 가설 C — 시야 컨트롤형(20%)을 협동 메타 콘텐츠와 연계
- **관찰**: vision z = +1.34 만 두드러진 균형형. SUPPORT 비중이 약간 높음(21.9%). 승률 52% — 안정적이지만 캐리는 아님.
- **콘텐츠 액션**: 와드/시야 기반 미션, 협동 미션, 정찰 보상. ARAM·정글캠프 챌린지 같은 사이드 모드 잘 받음.
- **리텐션 액션**: "정찰 마스터" 같은 보조형 챔프 진척 시스템. 듀오/플렉스 큐 친화.

## 7. 한계와 후속 분석 제언

- **식별자 부재**: 같은 플레이어를 여러 게임에 걸쳐 누적할 수 없음. PHASE 5(Riot API 수집)에서 puuid 기반 데이터를 모으면 "플레이어별 평균 프로필"로 군집화 → 진짜 페르소나가 가능.
- **승률과 행동 상관**: GPM·DPM·KP가 win과 자연 상관 → C2/C3는 결과의 반영이 일부 포함. 더 깨끗한 분리는 "고승률 게임 안에서만"으로 한 번 더 잘라보는 후속 분석으로 다룰 수 있음.
- **K=4 선택의 약점**: silhouette 점수 자체는 K=2부터 단조 감소. K=5~6도 시도해볼 가치 있지만, 4개 페르소나가 게임 운영 액션과 매핑하기에 가장 명료해서 채택.
